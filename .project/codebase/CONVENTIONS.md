## Function Design

**Size:**
- Functions range from 1-line utilities (`js/utils.js`) to ~80-line update/render functions (`js/main.js`)
- Rendering functions are the longest, containing sequential canvas draw calls

**Parameters:**
- Canvas context always passed as `ctx` (first parameter for draw functions)
- Camera passed as a plain `{ x, y }` object
- Delta time passed as `dt`
- `time` is elapsed game time (for animations)

**Return Values:**
- Generator functions return data structures: `generateSystem()` returns `bodies[]`
- Draw functions return nothing (void)
- Utility functions return computed values
- `consume*` pattern returns the value and resets internal state (see `js/input.js` `consumeClick()`, `consumeInteract()`)

**Coordinate Convention:**
- World coordinates: `x`, `y` (the body/ship position in world space)
- Screen coordinates: `sx`, `sy` (computed as `worldPos - camera + canvas/2`)
- This transform appears repeatedly — always inline, never abstracted
