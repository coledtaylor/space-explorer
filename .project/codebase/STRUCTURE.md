## Naming Conventions

**Files:**
- Scene files: PascalCase matching class name, e.g., `FlightScene.ts`, `BootScene.ts`
- Object files: PascalCase matching class name, e.g., `Ship.ts`
- Entry point: lowercase `main.ts`
- Config files: lowercase with dots, e.g., `vite.config.ts`, `tsconfig.json`

**Directories:**
- All lowercase: `src/`, `scenes/`, `objects/`
- Plural nouns for collections: `scenes/`, `objects/`

**Classes:**
- PascalCase, suffixed by category: `FlightScene`, `BootScene`, `Ship`
- Scene classes always end with `Scene`

**Exports:**
- One primary class export per file (named export, not default)
- Interfaces co-located with the class that owns them (e.g., `ScreenBounds` in `Ship.ts`)

## Where to Add New Code

**New Scene (game screen/state):**
- Create `src/scenes/{Name}Scene.ts`
- Export a class extending `Phaser.Scene` with a unique `key` string
- Register the scene in the `scene` array in `src/main.ts`

**New Game Object (entity):**
- Create `src/objects/{Name}.ts`
- Export a class extending an appropriate `Phaser.GameObjects.*` base class
- Import and instantiate from the relevant scene's `create()` method

**New Library/Utility Module:**
- Create `src/lib/{name}.ts` (lowercase, descriptive name)
- Use pure functions with named exports; prefer stateless utilities
- Note: The `src/lib/` directory does not currently exist on this branch but was used historically for physics, orbit math, units, etc.

**New Type Definitions:**
- Create `src/types/index.ts` or `src/types/{domain}.ts`
- Note: The `src/types/` directory does not currently exist on this branch but was used historically

**New Constants/Config:**
- For module-scoped constants: define at the top of the file that uses them (see `Ship.ts` pattern with `THRUST_ACCELERATION`, `ROTATION_SPEED`)
- For shared constants: create `src/lib/scaleConfig.ts` or similar (historical pattern)
