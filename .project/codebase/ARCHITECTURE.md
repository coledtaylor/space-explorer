## Data Flow

**Game Loop (per frame):**

1. Phaser calls `FlightScene.update(time, delta)` each frame
2. `FlightScene` reads keyboard input state from cached `Phaser.Input.Keyboard.Key` references
3. Based on input, `FlightScene` calls `Ship` methods: `applyThrust(delta)`, `rotateLeft(delta)`, `rotateRight(delta)`
4. `FlightScene` calls `ship.updatePosition(delta, bounds)` to integrate velocity into position with screen clamping
5. `FlightScene` reads `ship.speed` and updates HUD text

**Input Handling:**
- Keyboard keys (W, A, S, D) are registered in `FlightScene.create()` via `this.input.keyboard!.addKey()`
- Input is polled each frame in `update()` using `.isDown` checks
- No event-driven input handling; purely poll-based

**State Management:**
- All game state lives on object instances (Ship velocity, heading, position)
- No external state management library or global state store
- Ship owns its own physics state (`vx`, `vy`, `heading`) as private fields
- Scene owns references to its game objects and HUD elements
