## Data Flow

**Game Loop (per frame):**

1. `requestAnimationFrame` calls `loop(timestamp)` in `js/main.js`
2. Delta time (`dt`) is computed and clamped to 0.05s max
3. `update(dt)` runs: input aim updated, ship physics applied, orbits updated, HUD text set, click/interact checks, fuel regen near stars, system transition check
4. `render()` runs: canvas cleared, starfield drawn, camera hack set, bodies sorted and drawn, ship drawn, minimap drawn

**System Transition:**

1. Ship distance from origin exceeds 2000 units (checked in `update()`)
2. `currentSystemSeed` incremented by 1
3. `generateSystem(newSeed)` produces a fresh `bodies[]` array
4. Ship position reset to `(0, -300)`, velocity reduced to 30%
5. Camera snapped to new ship position, scan panel hidden

**Input Flow:**

1. Browser events set boolean flags on `Input` instance (`up`, `down`, `left`, `right`)
2. Click and interact are "consumable" — `consumeClick()` and `consumeInteract()` return the value once then reset to null/false
3. `Ship.update(dt, input)` reads directional flags directly from the `Input` instance
4. `main.js` calls `input.updateAim(shipScreenX, shipScreenY)` each frame to compute aim angle from mouse position

**Procedural Generation:**

1. `seededRandom(seed)` in `js/utils.js` returns a deterministic RNG function using the LCG algorithm (multiplier 16807, modulus 2^31-1)
2. `generateSystem(seed)` uses that RNG to create: 1 star (random type/color), 3-7 planets (random orbits/types), optional anomaly (30% chance)
3. All bodies are plain objects with a `kind` field (`'star'`, `'planet'`, `'anomaly'`) — no class hierarchy
4. Each body carries: position (`x`, `y`), `radius`, `color`, `name`, `description`, `details` object, and for planets: `orbitRadius`, `orbitAngle`, `orbitSpeed`

## Error Handling

**Strategy:** No explicit error handling exists. The codebase relies on:

**Patterns:**
- Defensive math: `dt` is clamped to 0.05s max to prevent physics explosions from tab-switching (`js/main.js` line 69)
- Null-safe rendering: `drawBody()` skips off-screen bodies early (`js/renderer.js` lines 7-8)
- Fuel clamping: `Math.max(0, ...)` and `Math.min(100, ...)` prevent fuel from going out of range (`js/ship.js` line 37, `js/main.js` line 138)
- Speed clamping: velocity is normalized when exceeding `maxSpeed` (`js/ship.js` lines 46-49)
- No try/catch blocks, no error boundaries, no user-facing error messages
